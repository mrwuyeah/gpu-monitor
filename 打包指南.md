# GPU监控应用打包指南（2026/5/22 多VLLM进程监控版本）

## 📦 打包方案概述

本指南介绍如何将 GPU 监控程序打包成可执行文件，支持 Windows/Linux 直接运行。

- 打包工具：PyInstaller
- 打包入口：`app.py`
- 运行依赖：目标机器需可执行 `nvidia-smi`

> 说明：本项目已有 `build_windows.bat` / `build_linux.sh` 一键脚本；也可以使用 `.spec` 文件打包。

## 💻 系统要求

### Windows 10/11
- Python 3.8+
- pip

### Ubuntu 24.04.x / 其它 Linux
- Python 3.10+
- pip
- NVIDIA 驱动（用于 GPU 监控）

## 🔨 打包步骤

### 方案A：在 Windows 上打包

#### 1. 安装依赖
在项目根目录执行：
```bash
python -m pip install -r requirements.txt
python -m pip install pyinstaller
```

#### 2. 一键打包
```bash
build_windows.bat
```

#### 3. 输出
- `dist\\gpu_monitor.exe`

### 方案B：在 Ubuntu/Linux 上打包

#### 1. 安装依赖
```bash
python3 -m pip install -r requirements.txt
python3 -m pip install pyinstaller
```

#### 2. 一键打包
```bash
chmod +x build_linux.sh
./build_linux.sh
```

#### 3. 输出
- `dist/gpu_monitor`

## 🧩 使用 .spec 文件打包（可选）

项目内存在多个 `.spec` 文件：
- `build_executable.spec` / `gpu_monitor.spec`：用于打包 Web 监控（包含 `templates/`）
- `monitor_gpu.spec`：用于打包 `monitor_gpu.py`（如你仍需要该脚本）

使用方式：
```bash
pyinstaller build_executable.spec
# 或
pyinstaller gpu_monitor.spec
```

## ✅ 打包资源说明

Web 页面模板位于 `templates/`，打包时需要作为数据文件加入。

- Windows：`--add-data "templates;templates"`
- Linux：`--add-data "templates:templates"`

一键脚本与 spec 已包含该配置。

## 🆘 常见问题

### Q1: 打包后运行提示找不到模板
确认打包时包含了 `templates/`（见上面的 add-data / spec 配置）。

### Q2: 运行时无法获取 GPU 信息
确认目标机器可以执行：
```bash
nvidia-smi
```

### Q3: 端口被占用
如果程序支持命令行参数（例如 `--port`），优先用参数启动；否则需要在源码中修改端口后重新打包。
